### 设计模式的目的

* 代码重用性（提取重复代码）
* 可读性（代码规范，便于阅读）
* 可扩展性（方便增加新功能）
* 可靠性（增加新功能，对以前的功能没有影响）
* 使程序呈现高内聚、低耦合的特性


### 设计模式的七大基本原则（design-principle）

* 单一职责原则

* 接口隔离原则

* 依赖倒置原则

* 里氏替换原则

* 开闭原则

* 迪米特法则

* 合成复用法则

>组合和聚合
